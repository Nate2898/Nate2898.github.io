Project Overview: Torea Travel Enterprise Network Solution

Project Goal

The objective was to design and implement a centrally managed, secure, and scalable IT infrastructure for Torea Travel's head office in Christchurch and its branch in Nelson. The solution was designed to support domestic and international travel operations while ensuring high availability and future-proofing for business expansion.

1. Business Case and Requirements

Torea Travel required a robust system to support a distributed workforce, including office-based agents and mobile users. Key requirements included:

  • Centralised Management: A server-based solution located at the Christchurch head office to manage both sites.
  • Core Services: Reliable internet access, email, file sharing, and an internal wiki/CMS.
  • Business Continuity: Comprehensive backup and disaster recovery plans to safeguard sensitive client data.
  • Connectivity: Secure VPN tunnels for site-to-site communication and remote access for mobile agents.

2. Technical Architecture

Network Infrastructure

The network was designed with a focus on High Availability (HA) to ensure minimal downtime.

  • Routing and Firewalls: The primary gateways utilised Netgate 4200 and 1100 appliances running pfSense. High availability was achieved through CARP (Common Address Redundancy Protocol) for IP redundancy and pfsync for state table synchronisation.
  • Connectivity: A peer-to-peer IPsec VPN connected the Christchurch and Nelson branches, while OpenVPN provided secure tunnels for remote employees.
  • Subnetting: A structured IPv4 plan was implemented (for example, 192.168.1.0/24 for Christchurch), with built-in scalability to add future branches.
Server and Software Stack

The environment utilised a mix of industry-standard enterprise software:

  • Operating Systems: Windows Server 2022 for core infrastructure and Windows 11 Pro for workstations.
  • Directory Services: Active Directory Domain Services (AD DS) was used for centralised identity and access management.
  • Network Services: Centralised DHCP with failover and resilient DNS were implemented to ensure continuous network operations.
  • Monitoring: An Xubuntu server running Prometheus and Grafana provided real-time traffic visualisation and system health monitoring.

3. Cloud Integration and Security

Hybrid Cloud Strategy

To enhance collaboration and data redundancy, the proposal integrated two primary cloud solutions:

  • Microsoft OneDrive: Provided 1TB of storage per user, allowing real-time collaboration via Microsoft 365.
  • Microsoft Azure: Utilised for Infrastructure as a Service (IaaS), providing scalable virtual machines and offsite backup storage via the Azure MARS agent.
Security and Compliance

A multi-layered security plan was established to mitigate cyber threats:

  • Firewall Policies: Layer 3 filtering on pfSense to block malicious traffic and manage DNS redirection.
  • Data Protection: Implementation of SSL/TLS protocols for data in transit and multi-factor authentication for all user accounts.
  • Ransomware Mitigation: A robust backup schedule utilising Veeam Backup and Replication, combining local storage with immutable cloud backups.

4. Implementation Roadmap

The project was designed to be executed over a six-week timeframe:

  • Phase 1-2: Requirement analysis, network design, and equipment procurement.
  • Phase 3: Hardware configuration, including pfSense HA setup and Windows Server deployment.
  • Phase 4-5: Rigorous testing of security protocols and network performance, followed by staff training.
  • Phase 6: Final documentation handover and client review.

5. Key Deliverables

  • Detailed Technical Report: Comprehensive documentation covering logical/physical diagrams, IP schemas, and security policies.
  • Functional Test-Bed: A simulated environment hosted on VMware vSphere to demonstrate server functionality, VPN connectivity, and failover capabilities.
  • Budget and Procurement Plan: An itemised hardware and software list totalling approximately $13,972 NZD, optimised for performance and future growth.

Project Overview: InterTrode Network Domain Design

The core objective of this project was to design and implement a robust, secure, and scalable network infrastructure for InterTrode, a small company requiring centralised management and enhanced security. The solution leverages a virtualised environment to host domain services, security appliances, and client workstations.

Technical Architecture and Infrastructure

The network is built upon a foundation of Windows Server 2022 and Hyper-V virtualisation.

Domain Controllers

The logical design features a single root domain, InterTrode.local, supported by two Domain Controllers (DC-1 and DC-2) hosted on separate physical machines to ensure redundancy.

Gateway and Security

A Sophos Firewall VM serves as the primary gateway, managing WAN/LAN interfaces and providing web filtering policies to differentiate access levels between servers and clients.

Hardware

The implementation utilised Dell Precision 3460 workstations (Intel Core i5-12500, 16GB RAM) as host machines for the virtual environment.

Key Features and Implementation

1. Identity and Policy Management
  • Active Directory (AD DS): Centralised security boundary for network organisation and access control.
  • Group Policy Objects (GPOs): Used for granular control over the environment, including disabling Command Prompt access for employees and managing automated Windows Server Update Service (WSUS) schedules.
  • Organisational Units (OUs): Logical containers were created for Admins, Employees, and Domain Controllers to streamline management.
2. Network Security and Encryption
  • IPsec Implementation: Secure, encrypted communication was established between the primary DC and specific client machines using Kerberos V5 authentication. The connection was verified through ESP (Encapsulating Security Payload) packet inspection.
  • Firewall Integration: The Sophos Firewall was joined to the Active Directory domain, allowing for identity-based web policies and encrypted SSL/TLS communication for user authentication.
3. Data Resilience and Availability
  • Distributed File System (DFS): Configured to provide a unified namespace and replication for shared folders across both Domain Controllers.
  • Hyper-V Replication: To minimise downtime, the critical Sophos Firewall VM is replicated from its primary host to DC-2, allowing for rapid recovery in the event of hardware failure.
  • Backup Solutions: Integrated Windows Server Backup was used to safeguard critical data folders, with automated schedules sending backups to separate machines on the domain.
4. Monitoring and Remote Management
  • Paessler PRTG: Installed on the primary DC to monitor CPU usage, disk health, and server uptime, providing real-time alerts for network issues.
  • Windows Admin Center (RSAT): Enabled centralised remote management of all domain-joined machines from a single web-based interface.
  • AnyDesk: Implemented on the primary DC to facilitate high-performance remote access for administrators.

Methodology: Migration and Documentation

The project demonstrated a Physical-to-Virtual (P2V) migration using the Disk2vhd utility. This involved capturing a physical Windows 10 instance and successfully deploying it as a functional VM within the Hyper-V environment. Comprehensive "Step-by-Step" documentation was also produced to guide junior admins through routine tasks such as user onboarding, password resets, and account decommissioning.

Conclusion

By combining Active Directory, Sophos security, and Hyper-V redundancy, this project delivered a professional-grade network capable of supporting InterTrode's current business needs while providing a clear path for future expansion.
